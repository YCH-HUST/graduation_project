## 2.2 后端开发技术

### 2.2.1 Django 框架

本系统后端采用 **Django 5.2** 作为核心 Web 框架。Django 是一个高级 Python Web 框架，遵循 MTV（Model-Template-View）架构模式，具有以下特点：

- **快速开发**：Django 提供了丰富的内置功能，包括 ORM、管理后台、表单处理等，可以大幅缩短开发周期
- **安全性高**：内置防止 SQL 注入、XSS、CSRF 等常见 Web 安全漏洞的机制
- **可扩展性强**：通过应用（App）机制实现模块化开发，便于功能扩展和代码复用

### 2.2.2 Django REST Framework

**Django REST Framework (DRF) 3.16** 是一个强大的 RESTful API 开发框架，提供：

- **序列化器（Serializers）**：实现数据模型与 JSON 格式的相互转换
- **视图集（ViewSets）**：简化 CRUD 操作的实现
- **权限控制**：灵活的认证和权限系统
- **API 文档**：自动生成可交互的 API 文档

### 2.2.3 JWT 认证机制

系统采用 **djangorestframework-simplejwt 5.5** 实现 JSON Web Token (JWT) 认证机制：

- **无状态认证**：服务器不需要存储会话信息，便于水平扩展
- **跨域支持**：Token 通过 HTTP Header 传输，天然支持跨域请求
- **安全性**：采用非对称加密算法，支持 Token 过期和刷新机制

JWT 配置参数：
- Access Token 有效期：1 天
- Refresh Token 有效期：7 天
- 认证头格式：`Authorization: Bearer <token>`

### 2.2.4 MySQL 数据库

系统使用 **MySQL 8.0** 作为关系型数据库，通过 **mysqlclient 2.2** 驱动与 Django ORM 集成：

- **字符集**：utf8mb4，支持完整的 Unicode 字符（包括中文和 Emoji）
- **排序规则**：utf8mb4_unicode_ci
- **事务支持**：使用 InnoDB 存储引擎，支持 ACID 事务

### 2.2.5 计算机视觉与深度学习

系统集成了多种深度学习框架用于舌象图像分析：

| 库名称 | 版本 | 用途 |
|--------|------|------|
| PyTorch | 2.9.1 | 深度学习框架 |
| TorchVision | 0.24.1 | 计算机视觉工具包 |
| Ultralytics | 8.4.2 | YOLO 目标检测框架 |
| OpenCV | 4.12.0 | 图像处理库 |
| NumPy | 2.2.6 | 数值计算库 |
| SciPy | 1.17.0 | 科学计算库 |
| Pillow | 12.1.0 | 图像处理库 |

**YOLO (You Only Look Once)** 是一种先进的实时目标检测算法，本系统使用 YOLOv8 模型进行舌象特征检测，能够识别舌象的多种病理特征，包括：
- 黑苔、地图舌、紫苔
- 红舌黄厚腻苔、红舌厚腻苔、白舌厚腻苔

### 2.2.6 跨域资源共享 (CORS)

使用 **django-cors-headers 4.9** 处理跨域请求，允许前端应用从不同域名访问后端 API。

## 2.3 前端开发技术

### 2.3.1 Next.js 框架

系统前端采用 **Next.js 16.1** 框架，这是基于 React 的全栈 Web 框架，具有以下核心特性：

- **App Router**：基于文件系统的路由机制，支持嵌套布局和并行路由
- **服务端渲染 (SSR)**：首屏服务端渲染，提升 SEO 和加载性能
- **静态站点生成 (SSG)**：支持预渲染静态页面
- **API 路由**：内置 API 端点支持
- **自动代码分割**：按需加载，优化包体积

### 2.3.2 React 组件库

系统基于 **React 19.2** 构建用户界面，采用现代化的组件化开发模式：

- **函数式组件**：使用 Hooks API 管理组件状态
- **组合式 API**：通过 `useState`、`useEffect`、`useCallback` 等 Hooks 实现逻辑复用
- **严格模式**：启用 StrictMode 帮助发现潜在问题

### 2.3.3 TypeScript 类型系统

项目全面采用 **TypeScript 5.x** 进行开发，实现静态类型检查：

- **类型安全**：编译时类型检查，减少运行时错误
- **智能提示**：IDE 提供完整的代码补全和类型推断
- **接口定义**：统一定义 API 请求和响应的数据结构
- **编译目标**：ES2017，兼顾现代特性与浏览器兼容性

### 2.3.4 TailwindCSS 样式框架

采用 **TailwindCSS 4.x** 作为 CSS 框架，实现原子化 CSS 开发：

- **实用优先**：通过组合原子类快速构建界面
- **响应式设计**：内置断点系统，适配多种屏幕尺寸
- **暗色模式**：支持系统级和手动切换暗色主题
- **按需编译**：只打包使用到的样式，减少 CSS 体积

### 2.3.5 shadcn/ui 组件库

UI 组件基于 **shadcn/ui** 构建，集成 Radix UI 无障碍组件：

| 组件 | 版本 | 功能 |
|------|------|------|
| @radix-ui/react-dialog | 1.1.15 | 对话框/模态框 |
| @radix-ui/react-select | 2.2.6 | 下拉选择器 |
| @radix-ui/react-tabs | 1.1.13 | 选项卡 |
| @radix-ui/react-tooltip | 1.2.8 | 工具提示 |
| @radix-ui/react-popover | 1.1.15 | 弹出框 |

shadcn/ui 的优势：
- **完全可定制**：组件代码直接放入项目，可自由修改
- **无障碍访问**：基于 Radix UI，符合 WAI-ARIA 标准
- **轻量级**：按需引入，不增加额外包体积

### 2.3.6 状态管理

采用 **Zustand 5.0** 进行全局状态管理：

- **轻量级**：核心代码仅几 KB，无依赖
- **简洁 API**：相比 Redux，代码量减少 80%
- **TypeScript 友好**：完整的类型推断支持
- **持久化支持**：可配合 localStorage 实现状态持久化

### 2.3.7 HTTP 客户端

使用 **Axios 1.13** 作为 HTTP 客户端：

- **请求/响应拦截器**：统一处理认证头和错误响应
- **自动 JSON 转换**：自动序列化请求体和解析响应
- **超时控制**：可配置请求超时时间
- **取消请求**：支持取消进行中的请求

## 2.4 系统架构

### 2.4.1 整体架构

系统采用 **B/S（Browser/Server）架构**，前后端分离部署：

```
┌─────────────────────────────────────────────────────────────────┐
│                         客户端 (浏览器)                          │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  Next.js 前端应用                          │  │
│  │  React 组件 ──▶ Zustand 状态 ──▶ Axios API 调用           │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │ HTTP/HTTPS
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Django 后端服务                             │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  Django REST Framework API                                 │  │
│  │  ├── accounts (用户认证)                                   │  │
│  │  ├── cases (病例管理)                                      │  │
│  │  ├── pipeline (AI 流水线)                                  │  │
│  │  ├── adminops (管理操作)                                   │  │
│  │  └── notifications (通知系统)                              │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
           │                                    │
           ▼                                    ▼
┌─────────────────────┐             ┌─────────────────────────────┐
│   MySQL 数据库       │             │    AI 模型服务               │
│   - 用户数据         │             │    - YOLO 舌象检测           │
│   - 病例数据         │             │    - NLP 问诊解析            │
│   - 诊断记录         │             │    - 证候推理引擎            │
└─────────────────────┘             └─────────────────────────────┘
```

### 2.4.2 后端模块划分

| 模块名称 | 功能描述 |
|----------|----------|
| accounts | 用户认证与授权，包括登录、注册、JWT 管理 |
| cases | 病例的创建、查询、更新，诊断结果管理 |
| pipeline | AI 诊断流水线，包括图像处理、NLP 分析、证候推理 |
| audit | 审计日志，记录敏感操作 |
| adminops | 管理员操作，健康检查、数据治理 |
| notifications | 通知系统，消息推送与管理 |

### 2.4.3 前端路由结构

系统实现三端一体的设计，根据用户角色展示不同界面：

| 路由 | 角色 | 功能 |
|------|------|------|
| /login | 公开 | 用户登录/注册 |
| /patient/* | 患者 | 新建病例、查看诊断结果 |
| /doctor/* | 医生 | 待审列表、病例审核、患者管理 |
| /admin/* | 管理员 | 健康监控、用户管理、数据治理 |

## 2.5 开发规范

### 2.5.1 代码规范

- **Python**: 遵循 PEP 8 代码风格指南
- **TypeScript/JavaScript**: 使用 ESLint 进行代码检查
- **Git**: 使用语义化提交信息（Conventional Commits）

### 2.5.2 API 设计规范

- 遵循 RESTful API 设计原则
- 使用 JSON 格式进行数据交换
- 统一的错误响应格式
- JWT Bearer Token 认证
- 分页参数：`page` 和 `page_size`

### 2.5.3 安全措施

- HTTPS 加密传输
- JWT Token 过期机制
- 密码加密存储（Django 默认 PBKDF2）
- CORS 跨域策略
- SQL 注入防护（Django ORM）
- XSS 防护（React 自动转义）

## 2.6 本章小结

本章详细介绍了中医智能辅助诊疗系统的开发技术和开发环境。后端采用 Django + Django REST Framework + MySQL 技术栈，集成 PyTorch 和 YOLOv8 实现舌象智能分析；前端采用 Next.js + React + TypeScript + TailwindCSS 技术栈，使用 shadcn/ui 构建现代化用户界面。系统采用前后端分离架构，通过 RESTful API 进行数据交互，使用 JWT 实现安全认证，具有良好的可扩展性和可维护性。
